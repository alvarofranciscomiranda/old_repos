(Para limpar o que escrevi aqui):

  • O valor de uma variável básica que não está na base é 0. Se estiver na base, o seu valor é o valor do termo independente respetivo (última coluna). Uma variável está na base se a sua coluna tiver um 1 numa das linhas da base e nas restantes linhas (incluindo a final) tudo estiver a zeros.
  • O custo reduzido (num problema de maximização) indica qual seria o incremento necessário (ou decremento, se for um problema de minimização) no coeficiente dessa variável na função objetivo para esta passar a ser relevante o suficiente para ser incluida na solução ótima
  • O preço sombra é o valor unitário para o qual o lucro aumentaria se o valor de restrição aumentasse. Por cada unidade que a restrição aumenta, o lucro aumenta pelo valor da última linha intersetada com a coluna respetiva a essa variável.
  • Aparentemente quando uma restrição é “=” não se usa S (útil para a seguinte)
    • Adicionalmente, quando é “oposto” ao sentido do que estamos a fazer (>= a minimizar ou <= a maximizar), o S nessa restrição é subtraído em vez de somado :) (a confirmar mas parece ser o caso)
    • Também no 2Phase, quando se calcula o F a minimzar, se a constante estiver positiva meter negativa na tabela (apesar do resto manter o sinal :) - isto faz sentido pq equações, mas é banhada)

    • Se Fmin > 0 -> O valor ótimo de F é maior que zero
      • concluímos que o problema de PL não tem feasible solutions
    • Se Fmin = 0 -> O valor ótimo de F é igual a zero

      • Se estiverem na base e os coeficientes das variáveis não artificiais dessa linha forem zeros, a restrição é redundante e pode ser ignorada

Transporte

  • Determinar U e V
    • Arbitrar uma linha / col (U/V) a 0, escolhendo a que tem mais variáveis básicas. Continuar a atribuir valores de modo a que o custo seja = soma de U e V para essa célula.
  • Custo Mínimo:
    • Quando há uma linha ou coluna com 0s no custo unitário temos 2 opções:
      1. Ignorar e pôr no fim o que sobrar
      2. Começar por aí.
        • Desempate: maior custo da restante linha / coluna
  • Degenerescência:
    • Sendo m o nr de linhas da matriz e n o nr de colunas da mesma, existe degenerescência quando o número de variáveis básicas (na base? a diferença ainda não é clara) é menor do que m+n-1. A solução considera-se degenerada, pois pelo menos uma das variáveis básicas é nula. Isto pode acontecer na definição da solução inicial possível ou na aplicação do algoritmo de transporte.
    • Solução: Entre as variáveis colocadas a zero, arbitrariamente escolher uma para ser utilizada como variável básica.
  • Solução ótima:
    • Chega-se à solução ótima quando para todo o i,j Δi,j>=0Δi,j>=0 (Δi,j=ci,j−vj−uiΔi,j=ci,j−vj−ui). Isto apenas é calculado para as variáveis não básicas (alocação atual = 0).
    • Se todo o Δi,jΔi,j para as variáveis não básicas for estritamente positivo, a solução é única. Se algum for 0, a solução continua a ser ótima mas não é única. Pode-se chegar a outra solução ótima usando essa célula com delta 0 como pivot (não sei se é pivot o nome disto, confirmar), visto que não terá efeito no custo da solução.

Afetação (Assignment)

Caso específico dos problemas de transporte.

Passos:

  1. Para cada linha subtriar a esta o elemento de menor custo (assim temos 0s pq esse elemento fica a 0)
  2. Ver se a solução é ótima: (NOTA: Ver “método dos zeros novo” abaixo, é o que o prof usava nas TPs)
    1. Encontrar número de linhas mínimo (nn) que cubram todos os zeros
      1. Escolher linha/coluna com 1 único 0
      2. Desenhar traço na coluna/linha (correspondência do passo anterior, trocada) que passe pelo 0
      3. Repetir até todos os 0s terem uma linha
    2. Se n=dimensão(matriz)n=dimensão(matriz), a solução é ótima
    3. Se for menor, continuar
  3. (semelhante ao passo 1) Para cada coluna, selecionar o elemento com menor custo e subtraí-lo a todos os elementos da coluna. Testar otimalidade (passo 2). Se não for, continuar
  4. Identificar o menor nr pelo qual não passam linhas (traços), chamado tt Atualizar a matriz:
    • Subtrair tt a todos os elementos pelos quais não passam linhas
    • Os elementos com só 1 linha atravessada ficam iguais
    • Aos elementos atravessados por 2 linhas, soma-se tt
    • (4.1) Verificar se a solução é ótima. Se não for, repetir 3

Método dos zeros novo

(Para fazer os traços para determinar se a solução é ótima)

  1. Em vez de usar linha/coluna com 1 zero, usar a linha/coluna que tem mais zeros (por cortar)
  2. Em caso de empate, ir pela dimensão (coluna/linha) com direções com mais zeros
    • Exemplo: 4 colunas / 3 linhas ambas com 3 zeros cada, escolher coluna neste caso

Identificação da solução ótima após a aplicação do método:

  • Escolher um grupo de NN zeros (com N=dimensão(matriz)N=dimensão(matriz)) em linhas e colunas diferentes
    • No final do algoritmo é sempre possível encontrar este grupo, senão o nr de linhas seria < dimensão(matriz) e portanto a solução não era ótima
  • A solução ótima é obtida marcando Xi,j=1Xi,j=1 nas posições dos 0s selecionados e Xi,j=0Xi,j=0 nas restantes posições.

Para determinar o valor da função objetivo (custo/tempo total), usamos a matriz original, somando os valores de custo da alocação escolhida.

Casos especiais / Restrições:

  • A matriz tem de ser quadrada
    • Adicionar “dummy” colunas e linhas (de zeros) no caso de nr de tarefas != nr de operários com coeficientes nulos na função objetivo
  • Um ou mais trabalhadores não podem ser atribuídos a algumas tarefas:
    • Atribuir valor infinito no valor correspondente da função objetivo
  • Um ou mais operários podem ser atribuídos a mais que uma tarefa:
    • Para cada trabalhador criar um número de linhas igual ao número de tarefas que pode fazer
  • Há coefs negativos na função objetivo
    • Somar o abosluto do coeficiente mais negativo a todos os elementos
  • Problemas de maxmização com coeficientes positivos:
    • Seja cc o maior coeficiente
    • Criar matriz com Coef′i,j=c−Coefi,jCoefi,j′=c−Coefi,j e minimizar

Outros factos:

  • A resolução de um problema de afetação pelo método simplex implica sempre a utilização do método de duas fases
  • Adicionar uma constante a todos os custos de afetação de um recurso não altera a afetação ótima do problema

Bottleneck Assignment Problem

  • Variante em que as tarefas são executadas simultaneamente
  • Minimizar o tempo da tarefa mais longa
  1. Escolher arbitrariamente uma solução possível e determinar o tempo TT da tarefa mais longa
  2. Definir uma matriz nova. Os elementos (xx) são 0 se x<Tx<T (estritamente menor) e 1 no caso oposto.
  3. Determinar o número NN de linhas cobrindo todos os 0s da matriz
    • Se N<dimensão(matriz)N<dimensão(matriz), solução ótima
    • Se N>dimensão(matriz)N>dimensão(matriz), não ótima
      • Escolher um outro conjunto de zeros, em linhas e colunas diferentes, tendo uma nova solução melhor que a anterior
      • Enquanto a solução não for ótima, repetir 2 e 3
